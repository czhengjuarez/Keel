Run pilots before full adoption.

## Pilot selection

Choose two sites:

- **Pilot A:** medium complexity marketing/documentation site.
- **Pilot B:** higher complexity product/dashboard site.

## Pilot success criteria

- No critical accessibility regressions.
- No critical visual regressions.
- Core journeys complete without UX regressions.
- Development velocity is equal or better than baseline.

## Rollout stages

1. **Stage 0:** internal QA only.
2. **Stage 1:** limited traffic or internal users.
3. **Stage 2:** full traffic for pilot properties.
4. **Stage 3:** org-wide migration starts.

## Exit criteria to full rebase

- Pilot success criteria met for both pilot sites.
- Release process and CI checks stable for two consecutive releases.
- Migration playbook validated and updated with real findings.
